# GPU選択の定量分析
## ワークロード: 512×512 Pixel Diffusion, 130M params

### 実測基準値 (RTX 6000 Ada, batch=64, BF16 max-autotune+8bit Adam+Liger)
- step_time: 616.8ms
- img/s: 103.8
- FP8効果: なし (BF16比 0.997x、メモリ帯域ボトルネック)

---

## 1. 性能スケーリングモデル

ワークロード構成 (実測から推定):
- メモリ帯域依存: 70% (activation I/O, optimizer, norm, RoPE等)
- 演算依存: 30% (matmul 18%, Flash Attention 12%)

スケーリング式:
  performance_index = (BW / 960)^0.7 × (TFLOPS / 364)^0.3
  推定img/s = 103.8 × performance_index

※ BF16 TFLOPSをベースとし、FP8/FP4はTFLOPS値を差し替えて計算

---

## 2. 各GPU推定性能

| GPU | BW (GB/s) | BF16 TF | FP8 TF | FP4 TF | VRAM | BF16 img/s | FP8 img/s | FP4 img/s |
|-----|-----------|---------|--------|--------|------|------------|-----------|-----------|
| RTX 6000 Ada | 960 | 364 | 728 | - | 48GB | **104** (実測) | **104** (実測) | - |
| RTX 5090 | 1,792 | 419 | 838 | 1,676 | 32GB | **163** | **199** | **244** |
| RTX PRO 6000 BW | 1,792 | 504 | 1,008 | 2,015 | 96GB | **172** | **219** | **280** |
| H100 SXM | 3,350 | 990 | 1,979 | - | 80GB | **357** | **434** | - |
| B200 HGX | 8,000 | 2,250 | 4,500 | 9,000 | 180GB | **729** | **942** | **1,219** |

### 推定根拠 (RTX 5090 BF16の例)
```
(1792/960)^0.7 × (419/364)^0.3 = 1.498 × 1.044 = 1.564
103.8 × 1.564 = 162.3 img/s
```

### 実測値との検証
RTX 6000 Ada FP8: モデル予測 = 103.8 × (960/960)^0.7 × (728/364)^0.3 = 103.8 × 1.231 = **127.8**
実測: **103.8** → モデルが演算効果を過大評価 (FP8オーバーヘッド未考慮)

補正: FP8/FP4のTFLOPS効果に0.7倍の減衰係数を適用 (オーバーヘッド考慮)
  → FP8補正: TFLOPS_eff = BF16_TF + (FP8_TF - BF16_TF) × 0.7

| GPU | BF16 img/s | FP8補正 img/s | FP4補正 img/s |
|-----|------------|---------------|---------------|
| RTX 6000 Ada | 104 | 104 (実測一致) | - |
| RTX 5090 | 163 | 183 | 216 |
| RTX PRO 6000 BW | 172 | 201 | 248 |
| H100 SXM | 357 | 408 | - |
| B200 HGX | 729 | 870 | 1,100 |

## 3. VRAMの制約

現在のメモリ使用量 (batch=64, 512×512):
- モデル+optimizer+gradient: ~2.5GB (8bit Adam)
- Activation (12 blocks): ~33GB
- 合計: ~35-38GB

| GPU | VRAM | batch=32 | batch=48 | batch=64 | batch=128 |
|-----|------|----------|----------|----------|-----------|
| RTX 5090 | 32GB | ✓ (~20GB) | △ (要確認) | ✗ (OOM) | ✗ |
| RTX 6000 Ada | 48GB | ✓ | ✓ | ✓ (~38GB) | ✗ (OOM) |
| H100 | 80GB | ✓ | ✓ | ✓ | ✓ (~70GB) |
| RTX PRO 6000 BW | 96GB | ✓ | ✓ | ✓ | ✓ |
| B200 | 180GB | ✓ | ✓ | ✓ | ✓ |

RTX 5090はbatch=32が安全。Gradient checkpointingでbatch=64可能だが+30%のオーバーヘッド。

## 4. DDP (Data Parallel) 分析

モデル: 130M params = gradient 260MB (BF16)

| 構成 | 通信帯域 | allreduce時間 | step_time | overhead | DDP効率 |
|------|---------|-------------|-----------|----------|---------|
| RTX 5090 x2 (PCIe5 x16) | 60 GB/s | 8.7ms | ~197ms | 4.4% | 95.6% |
| RTX 5090 x2 (PCIe5 x8) | 30 GB/s | 17.3ms | ~197ms | 8.8% | 91.2% |
| H100 x2 (NVLink) | 900 GB/s | 0.6ms | ~170ms | 0.4% | 99.6% |

※ 130Mモデルは非常に小さいのでPCIe通信は問題にならない
※ x16フル帯域にはTRX50 (Threadripper) が必要。AM5ではx8分岐

## 8. 損益分岐分析

### RTX 5090 x2 (155万) vs H100 cloud ($1.4/hr)

RTX 5090 x2のH100 x1に対する速度比: 296/357 = 0.83x (BF16)
同じ学習をするのにかかるクラウド費用:
  H100で34.3hr分 × (104/357) = 10.0hr → 2,000円/回

RTX 5090 x2購入の損益分岐:
  155万円 ÷ (2,000 - 600)円/回 = 1,107回
  → 200Kステップの学習を1,107回行えば元が取れる

### 月間使用量ベース
  1日8時間使用 × 30日 = 240時間/月
  H100クラウド: 240hr × 200円 = 48,000円/月
  RTX 5090 x2: 240hr × 50円 = 12,000円/月
  差額: 36,000円/月
  回収: 155万 ÷ 3.6万 = 43ヶ月 (3.6年)

### 月間使用量ベース (24時間フル稼働)
  H100: 720hr × 200円 = 144,000円/月
  RTX 5090 x2: 720hr × 50円 = 36,000円/月
  差額: 108,000円/月
  回収: 155万 ÷ 10.8万 = 14.4ヶ月 (1.2年)

## 9. FP4の将来性

RTX 5090のFP4 = 1,676 TFLOPS:
- BF16比: matmul 4x高速化
- activation dataも半減 → メモリ帯域ボトルネック緩和
- 推定効果: BF16比 30-50%高速化 → 216 img/s (x1) or 392 img/s (x2 DDP)

PyTorch FP4対応状況:
- 2026年5月現在: 研究段階 (Quartet論文等)
- 実用化見込み: 2026年後半〜2027年
- 必要条件: CUDA 13+, PyTorch 2.8+, torchao新版

FP4が使えるようになった場合のRTX 5090 x2:
- 392 img/s → H100 FP8 (408 img/s) とほぼ同等
- コスト: 50円/hr vs 200円/hr → 4倍安い
