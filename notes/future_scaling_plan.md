## FP8 vs FP4 の使い分け

### 核心ルール
FP4の効果は **matmulサイズ = tokens × D** で決まる。

```
D < 2048:  FP8が常に最適 (+9-14%)。FP4はオーバーヘッドで逆効果。
D ≥ 2048:  batchが十分なら FP4 > FP8。
D ≥ 3072:  FP4が圧勝 (1.62x vs 1.42x)。
```

### なぜDが重要か
FP4のHadamard transformはD次元ベクトルに対して16×16ブロックで動作。
- D=768: 48ブロック → オーバーヘッドが支配的
- D=2048: 128ブロック → matmul演算量がオーバーヘッドを上回る
- D=3072: 192ブロック → FP4のTensor Core活用率が最大

### batchも重要
同じD=2048でも:
- batch=32 (tokens=8192): FP4 1.06x (ほぼ効かない)
- batch=128 (tokens=32768): FP4 1.33x
- batch=256 (tokens=65536): FP4 1.42x

tokens × D の積が大きいほどFP4が有利。

## スケールアップ戦略

### 案1: D拡大 (patch=16維持)
```
現在:  patch=16, D=768,  depth=12 → 130M, seq=256
提案:  patch=16, D=2048, depth=20 → ~1.5B, seq=256

利点:
  - seqは同じ256 → attention計算量は変わらない
  - D=2048でFP4が有効 (1.42x)
  - 各トークンの表現力が2.7倍向上
  - GPU利用率が大幅向上 (10.5% → 30%+推定)

懸念:
  - パラメータ10倍 → 学習時間増
  - ただしFP4 1.42xとB300の45倍速で相殺可能
```

### 案2: JiT-XL相当 (patch=2, D=1152)
```
JiT-XL: patch=2, D=1152, depth=28 → 675M, seq=16384

利点:
  - JiT論文でFID 1.35達成の実績あり
  - ピクセル空間で最高品質

懸念:
  - seq=16384はattention O(seq²)が超重い
  - GPU効率が悪い (matmulが小さい)
  - FP4効かない (D=1152 < 2048)
  - 学習コスト: 案1の数十倍
```

### 案3: ハイブリッド (patch=16, D=2048, 大画像対応)
```
提案: patch=16, D=2048, depth=24 → ~2B

256×256: seq=256,  batch=256 → 高速学習
512×512: seq=1024, batch=64  → 高解像度
1024×1024: seq=4096, batch=16 → 超高解像度

利点:
  - 同一モデルで多解像度対応 (Patch Diffusion)
  - D=2048でFP4フル活用
  - JiT-XL (seq=16384) の1/4-1/64のattention計算量
  - GPU効率最大化

これが最もバランスが良い。
```

### 案1 vs 案2 vs 案3 のコスト比較 (B300)

```
                      params   seq    BF16 img/s   FP4 img/s   200Kステップ
案1 (D=2048 patch=16) ~1.5B    256    ~350         ~500         ~6.7時間
案2 (JiT-XL patch=2)  675M    16384   ~20          ~20(FP4効かない) ~170時間
案3 (D=2048 patch=16) ~2B      256    ~300         ~430         ~7.8時間
      同 512×512               1024   ~80          ~110         ~29時間
```

**案1/案3がコスト効率で圧勝。** JiT-XLのseq=16384は計算量が桁違いに多い。
